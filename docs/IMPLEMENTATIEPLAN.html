04

Multi-tenancy

Strategie

Row-Level Security (RLS)

PostgreSQL RLS met schema isolation per tenant. Elke query heeft verplicht een tenant_id. Middleware injecteert de tenant context automatisch.

Domeinen

Subdomain routing

*.adminyzr.io → tenant subdomains. Custom domeinen mogelijk via Cloudflare CNAME. White-label voor Sityzr: admin.klant.nl

// Tenant hiërarchie
Organization (tenant)
  ├── Users         // rollen: owner, admin, accountant, medewerker, viewer
  ├── Clients       // CRM: bedrijven + contactpersonen
  ├── Invoices      // verkoopfacturen
  ├── PurchaseInv   // inkoopfacturen
  ├── Projects      // projecten + milestones
  ├── TimeEntries   // urenregistratie
  ├── PunchCards    // strippenkaarten
  └── Settings      // branding, BTW, IBAN, factuurtemplate

11

Beveiliging

Transport & API

  • HTTPS everywhere — Let's Encrypt via Caddy
  • HSTS + CSP headers
  • Rate limiting op auth endpoints (Redis)
  • JWT short-lived (15 min access / 7 dagen refresh)
  • CORS whitelist per tenant
  • API keys hashed (bcrypt)

Data & Compliance

  • Payload RLS — tenant isolation op DB niveau
  • Soft deletes — geen data permanent verwijderd
  • Audit log — elke mutatie gelogd (wie, wat, wanneer)
  • GDPR — data export + verwijderrecht
  • BTW-wet — facturen onveranderbaar na versturen
  • AVG-proof logging — geen PII in logs

Aandachtspunten

⚖️
BTW-wetgeving: Facturen zijn juridische documenten — na versturen niet meer muteren. Alleen creditnota's toestaan. Nummering mag geen gaten hebben.
🏦
IBAN-validatie: Valideer IBAN bij inkoopfacturen. Phishing via valse facturen met gewijzigd IBAN is een reëel risico bij zakelijke betalingen.
👁️
OCR review: Altijd menselijke goedkeuring voor inkoopfacturen boven een instelbaar bedrag. Nooit volledig automatisch betaalbaar maken.
📊
Boekhoudkoppeling: Vraag je accountant welk format hij nodig heeft voor de BTW-aangifte. CSV/CAMT.053 of directe Snelstart/Twinfield koppeling?
💼
Salarisadministratie: Zeer complex (loonheffing, pensioen, Belastingdienst). Overweeg integratie via Nmbrs API in plaats van zelf bouwen.
🎫
Strippenkaarten: Definieer eenheden flexibel (uren, credits, taken, API-calls). Belangrijk voor hergebruik als SaaS module voor andere sectoren.
